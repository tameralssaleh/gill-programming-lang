# Contributing to GILL

Contributions are welcome.

## Ways to Contribute

- Report bugs
- Propose language features
- Improve interpreter architecture
- Expand Standard Library
- Improve documentation

## Before Submitting Changes

1. Open an issue describing the change (recommended for non-trivial work)
2. Discuss approach if it affects:
   - Syntax
   - Runtime model
   - Standard Library behavior

## Pull Request Guidelines

- Keep changes focused and minimal
- Include reasoning for design decisions
- Update docs when behavior changes
- Prototype features should go in `/proto`
- Production-direction work belongs in `/src`

## Code Priorities

1. Correctness
2. Clarity
3. Language consistency
4. Performance (later stage)

## Language Design Changes

Major changes should include:
- Motivation
- Examples
- Impact on existing syntax
- Runtime implications

## Questions / Ideas

Use Discussions for early proposals and open-ended design topics.
